# Contributing to Tomes Pass

Thank you for your interest in contributing to Tomes Pass! Your help improves the app and makes it safer for everyone.

## How to Contribute

1. Fork the repository.
2. Create a new branch for your change.
3. Commit your changes with a clear message.
4. Open a pull request against the `main` branch.

## Reporting Issues

If you find a bug or want to request a feature, please open an issue on GitHub.

Include:

- A descriptive title
- Steps to reproduce the issue
- Expected behavior vs actual behavior
- Your operating system and app version

## Submitting Pull Requests

- Keep changes focused and small.
- Explain why the change is needed in the PR description.
- Include screenshots or logs when relevant.
- Make sure your code is readable and well-formatted.

## Code Style

- Follow the existing project style in JavaScript and Markdown.
- Keep names descriptive and consistent.
- Use clear comments when the behavior is not obvious.

## Testing

- Test your changes before submitting.
- Verify the app still launches and key flows work.

## Security and Privacy

This project handles sensitive user data. Do not add features that expose passwords, PINs, or unencrypted vault data.

## Contact

For questions or help with contributions, open an issue or reach out via GitHub.
